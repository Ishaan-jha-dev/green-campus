PRODUCT OVERVIEW
----------------
GreenPulse Campus is a web-based sustainability intelligence platform for Indian
university campuses. It has three modules:
  1. CarbonLens — Scope 2 + 3 carbon emission tracker
  2. WasteAI — Camera-based waste classifier (in-browser AI)
  3. EnergyRadar — Smart meter anomaly detection

Design the full web app: Landing Page + Onboarding (3 steps) + Unified Dashboard
+ 3 module screens + Reports + Leaderboard + Settings.

================================================================================
SECTION 1 — AESTHETIC DIRECTION
================================================================================

VIBE
----
Confident Minimalism with Purposeful Data Density.
Think: Bloomberg Terminal built by a Scandinavian design studio.
Data-dense and credible, but clean enough that a 19-year-old opens it without
intimidation.

NOT a nature app. No leaf illustrations. No watercolour gradients.
NOT a generic SaaS dashboard. No purple/blue gradients. No Inter font.
NOT a government portal. No blue hyperlinks. No wall-of-tables.

DESIGN PRINCIPLES
-----------------
1. Credibility First — every number shows its source. No rounded-up values.
2. Progressive Disclosure — summary on dashboard, detail on click, raw data one
   more click. Complexity is hidden until requested, never absent.
3. Mobile Camera is Primary — WasteAI camera screen is one-handed, high contrast,
   designed for a dimly lit hostel corridor on a Redmi 10.
4. Earned Colour — Green = good/saved. Amber = warning. Rose = critical.
   Grey = neutral/inactive. Never break this colour mapping.

--text-display-lg     28px / 700 weight / line-height 1.25   Section headings
--text-heading-md     22px / 600 weight / line-height 1.35   Widget headers
--text-heading-sm     18px / 600 weight / line-height 1.40   Table headers, labels
--text-body-lg        16px / 400 weight / line-height 1.60   Primary body copy
--text-body-md        14px / 400 weight / line-height 1.55   Secondary, table cells
--text-caption        12px / 400 weight / line-height 1.50   Footnotes, timestamps
--text-label          11px / 600 weight / line-height 1.40   Uppercase tags, badges

MONO (JetBrains Mono — for all numbers)
--text-data-xl        32px / 700 weight   Big stat on dashboard cards
--text-data-lg        24px / 600 weight   Chart axis numbers, meter readings
--text-data-md        16px / 500 weight   Table data with numbers
--text-data-sm        12px / 400 weight   Inline data tags, CSV values

RULES
-----
- All data numbers (kWh, tCO2e, INR, %) use JetBrains Mono always.
- Letter-spacing on labels and badges: 0.08em. All other text: 0.
- Max line length: 68 characters body copy. Cards: 52ch max.
- Heading colour in UI screens: always --color-ink. Never coloured headings.

SECTION 16 — MOTION & INTERACTION
================================================================================

PRINCIPLES
----------
- Every animation communicates state change. No purely decorative motion.
- Enter: ease-out. Exit: ease-in. State change: ease-in-out.
- Spring physics only for confirmations and success moments.

DURATIONS
---------
Micro-interactions (button press, toggle):    100–200ms
Panel/drawer transitions:                     250–350ms
Page transitions:                             300–400ms
Loading loops:                                1,500ms cycle
Count-up number animations:                  600–900ms

KEY INTERACTIONS
----------------
Carbon number update
  Count-up animation 800ms / ease-out
  Number briefly pulses emerald on update (color transition 200ms)

Waste classification result reveal
  Result panel: translateY(100%) → 0 over 300ms / ease-out
  Dark overlay: fade in 200ms simultaneously

Heatmap render after CSV processing
  Cells fade in column by column (left to right)
  15ms stagger per column → total ~450ms for 30 days

Scan confirmation animation
  Emerald full-screen flash: opacity 0 → 0.4 → 0 over 300ms
  Panel bounce: scale 1 → 1.05 → 1 over 250ms

Report generation complete
  Card border animates from dashed mist → solid module color (clockwise sweep)
  Duration: 600ms / ease-in-out

Page transitions
  Fade + translateY(12px → 0) / 300ms / ease-out

Green Score gauge on first load
  Arc animates from 0 to score value / 1,200ms
  Easing: cubic-bezier(0.34, 1.56, 0.64, 1) — slight spring overshoot

Toast notifications
  Slide in from bottom-right: translateX(120%) → 0 / 250ms
  Auto-dismiss: 4 seconds with progress bar
  Mobile: swipe-right to dismiss

REDUCED MOTION (@media prefers-reduced-motion: reduce)
  All animations disabled. Numbers snap immediately.
  Heatmap cells appear together. Transitions: opacity only.

SECTION 17 — ACCESSIBILITY
================================================================================

COLOUR CONTRAST
---------------
Body text (ink on white):       18.1:1   — exceeds AAA
Mist on white (≥14px only):     4.54:1   — passes AA
Emerald on white (≥24px bold):  3.1:1    — passes AA for large text
Mint on Ink background:         7.2:1    — exceeds AAA

RULE: Never use mist (#6B7280) for text smaller than 14px.

KEYBOARD NAVIGATION
-------------------
All interactive elements reachable by Tab in logical reading order.
Skip-to-content link at top of every page (visually hidden until focused).
Camera screen: all buttons reachable via keyboard, Escape closes overlay.

FOCUS INDICATORS
----------------
Custom focus ring: 3px solid #059669, offset 2px.
Never remove outline. Override browser default with visible custom style.
Camera screen focus: white 3px ring (visible against dark bg).

SCREEN READERS
--------------
All Recharts graphs have visually hidden data table as fallback.
Chart containers: role='img' aria-label='[description of chart]'
Camera result: announced via aria-live='polite' region immediately on detection.
Override buttons: descriptive aria-labels, not just icon names.

TOUCH TARGETS
-------------
Minimum: 44×44px for all interactive elements.
Camera overlay buttons: 56×56px minimum.
Bottom nav items: full column width.
Leaderboard rows: full width tap target.

INDIAN-SPECIFIC NOTES
---------------------
Low bandwidth: WebP images + lazy loading. Total first load <350KB gzipped.
Charts render progressively (skeleton → data).

Budget Android devices:
  No CSS backdrop-filter (GPU-intensive).
  Max 2 box-shadows visible per screen.
  No grid animations on Android Chrome below v100.

Camera screen in bright sunlight:
  High-contrast mode toggle in Settings.
  viewfinder reticle: 3px stroke (not 2px) when high-contrast enabled.

Hindi readiness:
  All strings in /locales/en.json structure.
  Noto Sans Devanagari in font fallback stack.
  RTL layout not required (Hindi is LTR).
